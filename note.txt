Création et test d’une image Docker
1 - Création de l’image
mkdir webapp
cd webapp
vi Dockerfile
Copiez le contenu du Dockerfile dans le dossier webapp de ce dépôt

Récupérer le code :

git clone https://github.com/diranetafen/static-website-example.git

2 - Exécuter et tester l’image

Construire l’image :

docker build -t webapp:v1 .

Vérifier que l’image a bien été créée :

docker images

Démarrer un conteneur :

docker run --name webapp -d -p 80:80 webapp:v1

Accéder à l’application dans le navigateur avec l’adresse de la machine suivie du port 80 :

http://192.168.99.x:80

Gestion de l’image sur DockerHub

1 - Connexion à DockerHub
Identifiant : eazytraining
Mot de passe : eazytraining_dockerhub_password

Se connecter :

2 - Envoyer (push) l’image

Taguer l’image :

docker tag id_image eazytraining/webapp:v1

Envoyer l’image sur DockerHub :

docker push eazytraining/webapp:v1

3 - Vérification

Aller sur votre compte DockerHub
Cliquer sur Repositories
Vérifier le nom de l’image envoyée
